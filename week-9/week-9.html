PDSA - Week-9: Dynamic Programming


Dynamic programming

  • Solution to original problem can be derived by combining solutions to subproblems

    Examples: Factorial, Insertion sort, Fibonacci series

  • Anticipate the structure of subproblems

  • Derive from inductive definition

  • Solve subproblems in topological order

Memoization

  • Inductive solution generates same subproblem at different stages
  • Na√Øve recursive implementation evaluates each instance of subproblem from scratch
  • Build a table of values already computed ‚Äì Memory table
  • Store each newly computed value in a table
  • Look up the table before making a recursive call

 

Example of number in Fibonacci series:-

Simple recursive

def fibrec(n):
    if n <= 1:
        return n 
    return fibrec(n - 1) + fibrec(n - 2)

 

Memoization

memo ={}
def fib(n):
    if n <= 1:
        memo[n] = n
    if n not in memo:
        memo[n] = fib(n-1) + fib(n-2)
    return memo[n]

 

 

Dynamic programming

def fib(n):
    T = [0] * (n + 1)
    T[1] = 1
    for i in range(2, n + 1):
        T[i] = T[i - 1] + T[i - 2]
    return T[n]

Dynamic programming problems

Grid path Problem

  • Rectangular grid of one-way roads

  • Can only go up and right

  • How many paths from (0, 0) to ?

  • Every path has segments

  • What if an intersection is blocked?

  • Need to discard paths passing through blocked intersection

  • Inductive structure

  • Fill the grid by row, column or diagonal

Memoization vs dynamic programming

  • Barrier of holes just inside the border

  • Memoization never explores the shaded region

  • Memo table has O(m + n) entries

  • Dynamic programming blindly fills all mn cells of the table

  • Tradeoff between recursion and iteration

    • ‚ÄúWasteful‚Äù dynamic programming still better in general

 

  • Complexity is using dynamic programming, using memorization

 

Implementation for Grid Path problem:

Simple recursive

def grid_rec(x,y,barrier):
    global count
    count+=1
    if x < 1 or y < 1:
        return 1
    elif (x,y) in barrier:
        return 0       
    else:
        return grid_rec(x-1,y,barrier) + grid_rec(x,y-1,barrier)

Longest Common Sub Word (LCW)

  • Given two strings, find the (length of the) longest common sub word
  • Subproblems are LCW(i, j), for 0 ‚â§ ùëñ ‚â§ ùëö, 0 ‚â§ ùëó ‚â§ ùëõ
  • Table of ùëö + 1 ùëõ + 1 values
  • Inductive structure

 

  • Start at bottom right and fill row by row or column by column

     

     

Implementation

def LCW(s1,s2):
    import numpy as np
    (m,n) = (len(s1),len(s2))
    lcw = np.zeros((m+1,n+1))
    maxw = 0
    for c in range(n-1,-1,-1):
        for r in range(m-1,-1,-1):
            if s1[r] == s2[c]:
                lcw[r,c] = 1 + lcw[r+1,c+1]
            else:
                lcw[r,c] = 0
            if lcw[r,c] > maxw:
                maxw = lcw[r,c]                
    return maxw
s1 = 'bisect'
s2 = 'secret'
print(LCW(s1,s2))

Longest Common Subsequence (LCS)

  • Subsequence ‚Äì can drop some letters in between
  • Subproblems are LCS(i, j), for 0 ‚â§ ùëñ ‚â§ ùëö, 0 ‚â§ ùëó ‚â§ ùëõ
  • Table of ùëö + 1 ùëõ + 1 values
  • Inductive structure

 

  • Start at bottom right and fill row by row, column or diagonal

Implementation

def LCS(s1,s2):
    import numpy as np
    (m,n) = (len(s1),len(s2))
    lcs = np.zeros((m+1,n+1))
    for c in range(n-1,-1,-1):
        for r in range(m-1,-1,-1):
            if s1[r] == s2[c]:
                lcs[r,c] = 1 + lcs[r+1,c+1]
            else:
                lcs[r,c] = max(lcs[r+1,c], lcs[r,c+1])                
    return lcs[0,0]
s1 = 'secret'
s2 = 'bisect'
print(LCS(s1,s2))

Output

4.0

Edit distance

  • Minimum number of editing operations needed to transform one document to the other
  • Subproblems are ED(i, j), for 0 ‚â§ ùëñ ‚â§ ùëö, 0 ‚â§ ùëó ‚â§ ùëõ
  • Table of ùëö + 1 ùëõ + 1 values ‚ñ™
  • Inductive structure

  • Start at bottom right and fill row, column or diagonal

     

Implementation

def ED(u,v):
    import numpy as np
    (m,n) = (len(u),len(v))
    ed = np.zeros((m+1,n+1))
    for i in range(m-1,-1,-1):
        ed[i,n] = m-i
    for j in range(n-1,-1,-1):
        ed[m,j] = n-j
    for j in range(n-1,-1,-1):
        for i in range(m-1,-1,-1):
            if u[i] == v[j]:
                ed[i,j] = ed[i+1,j+1]
            else:
                ed[i,j] = 1 + min(ed[i+1,j+1], ed[i,j+1], ed[i+1,j])
    return(ed[0,0])
print(ED('bisect','secret'))

Matrix multiplication

  • Matrix multiplication is associative

  • Bracketing does not change answer but can affect the complexity

  • Find an optimal order to compute the product

  • Compute C ( i, j), 0 ‚â§ ùëñ, ùëó < ùëõ, only for ùëñ ‚â§ ùëó

  • C ( i, j), depends on C ( i, k ‚Äì 1) , C( k, j) for every ùëñ < ùëò ‚â§ ùëó

  • Diagonal entries are base case, fill matrix from main diagonal

Implementation

def MM(dim):
    n = dim.shape[0]
    C = np.zeros((n,n))
    for i in range(n):
        C[i,i] = 0
    for diff in range(1,n):
        for i in range(0,n-diff):
            j = i + diff
            C[i,j] = C[i,i] + C[i+1,j] + dim[i][0] * dim[i+1][0] * dim[j][1]
            print(C)
            for k in range(i+1, j+1):
                C[i,j] = min(C[i,j],C[i,k-1] + C[k,j] + dim[i][0] * dim[k][0] * dim[j][1])
            print(C)
    return(C[0,n-1])

def MM(dim):
    n = len(dim)
    C = []
    for i in range(n):
        L = []
        L=[0]*n
        C.append(L.copy())        
    for diff in range(1,n):
        for i in range(0,n-diff):
            j = i + diff
            KL = []
            for k in range(i, j):
                KL.append(C[i][k] + C[k+1][j] + dim[i][0] * dim[k][1] * dim[j][1])
            C[i][j] = min(KL)
    return(C[0][n-1])
